Undergraduate Dissertation: A Comparative Study of Evaluating Data Backup Techniques

A technical study comparing backup methods under different file structures and data patterns, with a focus on performance, practicality, and business relevance.

Research Problem

As data volumes continue to grow, organizations need backup methods that protect critical information efficiently while supporting business continuity. This dissertation explored how different backup techniques perform under different file structures, especially when the total amount of data remains the same but the way that data is organized changes.

Approach

The project compared full and incremental backup techniques through practical experimentation. Bash scripting on Linux was used to create and test demo backup systems, measuring performance across different backup scenarios, including large individual files and many smaller files with the same total data size.

Key Findings

The study found that incremental backup offered clear efficiency advantages in many situations, especially when data changes were repeated over time. It also showed that file structure matters: backing up many smaller files can create more overhead than backing up the same total size as fewer large files, which affects performance and backup strategy decisions.

Why It Matters

This dissertation reflects my early interest in data, system reliability, and practical technical evaluation. It is not only a theoretical comparison of backup concepts, but also a hands-on study that connects technical testing with real operational questions businesses face when protecting valuable data.
